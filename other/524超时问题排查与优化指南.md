# 524 超时问题排查与优化指南

## 问题描述

在导入文章功能中，当文章包含较多图片时，出现 **524 Origin Time-out** 错误，导入失败。

**错误特征**：

- 错误码：524 (Origin Time-out)
- 超时时间：约 33 秒
- 触发场景：导入包含多张图片的文章，开启图片处理功能

---

## 问题分析

### 超时链路

```
用户浏览器 → CDN/ESA → 外层 Nginx → 容器内 Nginx → 后端服务 → 图片源站
```

每一层都可能存在超时限制，需要逐层排查。

### 超时原因

1. **前端请求超时**：默认 15 秒
2. **CDN/ESA 超时**：阿里云 ESA 默认 30 秒
3. **外层 Nginx 超时**：默认 60 秒
4. **容器内 Nginx 超时**：默认 60 秒
5. **后端图片下载超时**：每张图片 60 秒
6. **图片源站访问慢**：GitHub raw 在国内访问缓慢

---

## 排查步骤

### 第一步：确认超时层级

#### 1.1 查看错误码

| 错误码 | 超时位置 | 说明 |
|:---:|:---:|:---:|
| 408 | 前端/浏览器 | 客户端请求超时 |
| 504 | Nginx | 网关超时 |
| 524 | CDN/ESA | 源站响应超时 |
| 500 | 后端服务 | 服务器内部错误 |

**本次问题**：524 错误，说明是 **CDN/ESA 层超时**。

#### 1.2 确认超时时间

- 观察到约 **33 秒**超时
- 阿里云 ESA 默认源站超时：**30 秒**
- 符合 ESA 超时特征（30秒 + 重试时间）

### 第二步：验证源站直连

**目的**：排除 CDN/ESA 影响，确认源站本身是否正常。

#### 2.1 获取源站 IP

```bash
# 查看 DNS 解析
nslookup yourdomain.com

# 或使用 ping
ping yourdomain.com
```

#### 2.2 直连测试

```bash
# 方式 1：使用 curl 直连源站 IP
curl -X POST http://源站IP/api/admin/articles/import \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files=@test.md" \
  -F "source_type=hexo" \
  -F "upload_images=true" \
  --max-time 600

# 方式 2：浏览器访问源站 IP
http://源站IP/admin
```

**结果判断**：
- ✅ 直连成功：确认是 CDN/ESA 超时问题
- ❌ 直连失败：需要优化后端逻辑

#### 2.2 优化图片下载并发逻辑

**文件**：`server/internal/service/article.go`

**优化点**：
1. 限制并发数为 10，避免资源耗尽
2. 使用 channel 收集结果，确保所有图片处理完成
3. 添加错误日志，失败的图片不影响整体导入

```go
// 并发下载上传图片（限制并发数为 10）
const maxConcurrency = 10
replacements := make(map[string]string)
resultChan := make(chan struct {
    oldURL string
    newURL string
    err    error
}, len(uniqueURLs))
sem := make(chan struct{}, maxConcurrency)

for url := range uniqueURLs {
    go func(imgURL string) {
        sem <- struct{}{}
        defer func() { <-sem }()

        newURL, err := s.downloadAndUploadSingleImage(ctx, imgURL, host, imageProxy)
        resultChan <- struct {
            oldURL string
            newURL string
            err    error
        }{imgURL, newURL, err}
    }(url)
}

// 收集结果
processedCount := 0
totalCount := len(uniqueURLs)
for processedCount < totalCount {
    result := <-resultChan
    processedCount++
    if result.err == nil {
        replacements[result.oldURL] = result.newURL
    } else {
        logger.Warn("图片处理失败 (%s): %v", result.oldURL, result.err)
    }
}
```

**修改后**：
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # 超时配置：支持文章导入等耗时操作
    proxy_connect_timeout 600s;
    proxy_send_timeout 600s;
    proxy_read_timeout 600s;
}
```

**参数说明**：
- `proxy_connect_timeout`：与后端建立连接的超时时间
- `proxy_send_timeout`：向后端发送数据的超时时间
- `proxy_read_timeout`：从后端读取响应的超时时间

### 方案 5：配置阿里云 ESA 超时（重点）

**问题根源**：阿里云 ESA 默认源站超时为 30 秒，这是导致 524 错误的主要原因。

#### 5.1 登录阿里云 ESA 控制台

访问：https://esa.console.aliyun.com/

#### 5.2 配置回源规则

##### 方法 1：全局回源超时配置

1. 进入 **ESA 控制台**
2. 选择您的站点（如 `jerion.cn`）
3. 左侧菜单找到 **"源站设置"** 或 **"回源配置"**
4. 调整以下参数：

| 参数 | 默认值 | 推荐值 | 说明 |
|:---:|:---:|:---:|:---:|
| 源站连接超时 | 30s | 60s | 与源站建立连接的超时时间 |
| 源站读取超时 | 30s | 600s | 从源站读取响应的超时时间 |
| 源站写入超时 | 30s | 600s | 向源站发送请求的超时时间 |

##### 方法 2：针对特定路径配置（推荐）

如果 ESA 支持路径规则，可以单独为导入接口设置长超时：

1. 进入 **ESA 控制台**
2. 选择您的站点
3. 左侧菜单找到 **"回源规则"**
4. 点击 **"新增规则"**

**规则配置示例**：

```yaml
规则名称: 文章导入接口超时配置
规则内容:
  - 传入请求类型: 
    - 自定义规则:
      - 匹配字段: URI 路径
      - 运算符: 包含
      - 匹配值: /api/v1/admin/articles/import
      - 变换处理: 不区分大小写
  - 则执行:
    - 回源超时时间: 300 秒（最高）
```

##### 方法 3：通过 API 配置（高级）

如果控制台找不到配置项，可以提交工单或使用 API：

```bash
# 使用阿里云 CLI 配置
aliyun esa UpdateSiteOriginTimeout \
  --SiteId 110567533884465 \
  --ConnectTimeout 60 \
  --ReadTimeout 600 \
  --WriteTimeout 600
```

#### 5.3 验证配置生效

配置完成后，等待 1-2 分钟让配置生效，然后测试：

```bash
# 测试导入接口
curl -X POST https://jerion.cn/api/admin/articles/import \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files=@test.md" \
  -F "source_type=hexo" \
  -F "upload_images=true" \
  --max-time 600 \
  -w "\nTime: %{time_total}s\n"
```

**预期结果**：
- ✅ 请求成功，耗时 > 30 秒但 < 600 秒
- ❌ 仍然 524 错误：检查其他层级配置

#### 5.4 ESA 配置注意事项

1. **免费版限制**：
   - 部分 ESA 免费版可能不支持自定义超时
   - 如果找不到配置项，需要升级套餐或联系技术支持

2. **配置优先级**：
   - 路径规则 > 全局配置
   - 建议使用路径规则，避免影响其他接口

3. **缓存影响**：
   - POST 请求默认不缓存，超时配置直接生效
   - GET 请求可能受缓存影响，需要清除缓存

4. **监控告警**：
   - 在 ESA 控制台配置超时告警
   - 及时发现超时问题

### 方案 7：优化图片下载速度（治本）

**问题根源**：GitHub raw 图片在国内访问缓慢，导致下载超时。

#### 7.1 使用图片代理

**前端配置**：

在导入文章时，勾选"图片处理"后，填写代理地址：

```
图片代理地址: https://gh.llkk.cc/
```

**后端实现**：

自动检测 `raw.githubusercontent.com` 并添加代理前缀：

```go
// 应用图片代理（如果配置了代理地址）
downloadURL := imgURL
if imageProxy != "" {
    proxy := strings.TrimSpace(imageProxy)
    if !strings.HasSuffix(proxy, "/") {
        proxy += "/"
    }
    // 为 GitHub raw 地址添加代理前缀
    if strings.Contains(imgURL, "raw.githubusercontent.com") {
        downloadURL = proxy + imgURL
    }
}
```

**效果**：
- 原始地址：`https://raw.githubusercontent.com/user/repo/main/image.png`
- 代理地址：`https://gh.llkk.cc/https://raw.githubusercontent.com/user/repo/main/image.png`
- 下载速度：从 10+ 秒降低到 1-2 秒

## 故障排查

### 问题 1：仍然 524 错误

**可能原因**：
1. ESA 配置未生效（需等待 1-2 分钟）
2. ESA 免费版不支持自定义超时
3. 存在其他中间层（如 Cloudflare）

**排查步骤**：
1. 直连源站 IP 测试
2. 检查 ESA 配置是否保存成功
3. 联系阿里云技术支持

### 问题 2：部分图片下载失败

**可能原因**：
1. 图片源站不稳定
2. 图片代理服务失效
3. 单张图片超时（> 300 秒）

**解决方案**：
1. 更换图片代理服务
2. 延长单张图片超时时间
3. 跳过失败图片，继续导入

### 问题 3：导入成功但图片未上传

**可能原因**：
1. 未勾选"图片处理"
2. 图片 URL 格式不支持
3. 文件服务配置错误

**排查步骤**：
1. 检查前端是否勾选"图片处理"
2. 查看后端日志：`docker logs app-container`
3. 检查文件服务配置（OSS/本地存储）
