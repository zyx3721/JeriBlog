## 📋 版本概述

JeriBlog v1.0.1 主要围绕 **RSS 订阅功能** 进行了全面优化和增强，新增了文章变更检测、删除恢复机制、重复文章自动清理等核心功能，显著提升了 RSS 订阅管理的智能化水平。

---

## ✨ 新增特性

### RSS 订阅功能增强

- **文章变更检测系统**
  - 支持检测标题、链接、发布时间三个维度的变更
  - 前端显示变更标签（标题已更新、链接已更新、发布时间已更新）
  - 可同时显示多个变更标签
  - 标记已读后自动清空变更标签
  - 新增 `update_type` 字段记录变更类型（逗号分隔）

- **文章删除与恢复机制**
  - 新增 `is_deleted` 字段支持软删除
  - RSS 源中移除的文章自动标记为已删除
  - RSS 源中恢复的文章自动恢复为未读状态
  - 支持按删除状态筛选文章

- **RSS 订阅管理功能**
  - 新增"立即刷新 RSS"按钮（手动触发抓取所有订阅源）
  - 支持关键词搜索文章标题
  - 支持按状态筛选（未读/已读/已删除）
  - 支持按友链来源筛选
  - 优化列表布局和居中显示

- **重复文章自动清理**
  - 标题和链接都变化时自动删除旧文章
  - 文章更新后自动检查并删除重复项
  - 优先保留链接匹配的文章（链接作为更稳定的标识符）
  - 避免同一篇文章因标题/链接反复变化而重复出现

- **友链 RSS 地址变更处理**
  - 友链 RSS 地址变更时自动清理该友链的所有旧文章
  - 新增 `DeleteByFriendID` 方法支持批量删除

## 🔄 API 变更

### 新增接口

- `POST /api/v1/admin/rssfeed/refresh` - 立即刷新 RSS 订阅源（需超级管理员权限）

### 接口优化

- `GET /api/v1/admin/rssfeed` - 新增搜索和筛选参数支持
- `PUT /api/v1/admin/rssfeed/:id/read` - 标记已读时清空 `update_type` 字段
- `PUT /api/v1/admin/rssfeed/read-all` - 批量标记已读时清空 `update_type` 字段

## 🔄 升级说明

### 数据库迁移

从 v1.0.0 升级到 v1.0.1 需要执行以下数据库变更：

```sql
-- 1. 移除 description 字段
ALTER TABLE rss_articles DROP COLUMN IF EXISTS description;

-- 2. 新增 is_deleted 字段
ALTER TABLE rss_articles ADD COLUMN IF NOT EXISTS is_deleted BOOLEAN DEFAULT false;

-- 3. 新增 update_type 字段
ALTER TABLE rss_articles ADD COLUMN IF NOT EXISTS update_type VARCHAR(50) DEFAULT '';

-- 4. 移除唯一索引约束
DROP INDEX IF EXISTS idx_friend_title;
DROP INDEX IF EXISTS idx_friend_link;

-- 5. 创建普通索引
CREATE INDEX IF NOT EXISTS idx_friend_title ON rss_articles(friend_id, title);
CREATE INDEX IF NOT EXISTS idx_friend_link ON rss_articles(friend_id, link);
```

### 注意事项

- 升级后首次刷新 RSS 订阅源时，会自动检测并清理重复文章
- 已删除的文章会被标记为 `is_deleted = true`，不会物理删除
- 文章变更检测仅对升级后新抓取的文章生效
