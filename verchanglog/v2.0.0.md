## 📋 版本概述

JeriBlog v2.0.0 是一个里程碑式的重大更新版本，围绕 **文件管理系统**、**通知消息系统**、**反馈投诉系统**、**动态管理系统** 进行了全面重构和功能增强。新增了文件引用计数机制、文件选择器组件、通知清空功能、反馈邮件通知、动态视频支持等核心功能，并完成了移动端响应式优化，显著提升了系统的易用性和稳定性。

## ✨ 新增特性

### 文件管理系统重构

- **文件引用计数机制**
  - 新增 `reference_count` 字段记录文件被引用次数
  - 文件上传时自动初始化引用计数为 0
  - 文件被引用时自动增加计数，取消引用时自动减少计数
  - 支持文件引用详情查询，显示所有引用位置
  - 引用详情包含：文章、用户头像、系统设置、友链、动态、评论、菜单、反馈投诉
  - 文件列表显示引用计数和引用类型标签
  - 支持智能跳转到引用位置（新窗口打开）

- **文件选择器组件**
  - 统一的文件选择器组件，支持从文件管理中选择已上传文件
  - 支持单选和多选模式
  - 支持按文件类型过滤（图片、视频、音频、文档等）
  - 支持按扩展名过滤（如 `.jpg,.png,.gif`）
  - 支持拖拽上传（反馈投诉表单）
  - 集成到用户管理、系统设置、博客设置、友链管理、动态管理等模块

- **文件用途分类优化**
  - 新增动态相关类型：`moment-image`（动态配图）、`moment-video`（动态视频）、`moment-music`（动态音乐）
  - 新增反馈相关类型：`feedback-attachment`（反馈附件）
  - 支持按用途筛选文件（模糊匹配）
  - 文件列表显示用途标签

- **文件状态管理优化**
  - 修复系统设置更换图片时误标记被引用文件为未使用的问题
  - 修复上传失败后切换页面保存配置时的状态残留问题
  - 修复文件删除时引用计数未减少的问题
  - 优化文件状态判断逻辑

### 通知消息系统增强

- **附件上传优化**

## 🔄 升级说明

### 数据库迁移

从 v1.0.1 升级到 v2.0.0 需要执行以下数据库变更：

```sql
-- 1. 新增文件引用计数字段
ALTER TABLE files ADD COLUMN IF NOT EXISTS reference_count INT DEFAULT 0;

-- 2. 创建索引
CREATE INDEX IF NOT EXISTS idx_files_reference_count ON files(reference_count);

-- 3. 初始化现有文件的引用计数（需要根据实际引用情况计算）
-- 注意：此步骤需要手动执行，根据实际引用情况更新引用计数
-- 示例：
-- UPDATE files SET reference_count = (
--   SELECT COUNT(*) FROM articles WHERE cover_image = files.url
--   + SELECT COUNT(*) FROM users WHERE avatar = files.url
--   + ...
-- );
```

### 注意事项

- 升级后需要手动初始化现有文件的引用计数
- 文件引用计数机制仅对升级后新上传的文件自动生效
- 建议在升级前备份数据库
- 升级后首次使用文件选择器时，可能需要刷新文件列表
- 通知清空功能仅删除已读消息，保留未读消息
- 反馈投诉回复时会自动发送邮件通知，需确保邮件服务配置正确
