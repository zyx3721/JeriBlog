# 常见问题（FAQ）

本页面收集了 JeriBlog 使用过程中的常见问题和解决方案。

## 安装部署

### Q: 如何安装 JeriBlog？

A: 请参考项目 README 文档中的安装说明。基本步骤：

1. 克隆项目代码
2. 安装依赖
3. 配置数据库
4. 配置环境变量
5. 启动服务

### Q: 支持哪些数据库？

A: JeriBlog 支持 MySQL、PostgreSQL 等主流关系型数据库。

### Q: 如何配置 HTTPS？

A: 建议使用 Nginx 反向代理配置 SSL 证书。具体配置方法请参考 Nginx 官方文档。

## 登录认证

### Q: 忘记管理员密码怎么办？

A: 可以通过以下方式重置密码：

1. 直接修改数据库中的用户密码（需要加密）
2. 使用密码重置功能（如果已配置邮箱）
3. 联系系统管理员

### Q: Token 过期怎么办？

A: 系统会自动使用 `refresh_token` 刷新 `access_token`。如果刷新失败，需要重新登录。

### Q: 支持哪些第三方登录？

A: 目前支持 GitHub、Google 等 OAuth 登录。需要在 [OAuth 配置](OAuth-Config) 中配置相应的 Client ID 和 Secret。

## 文章管理

### Q: 文章保存后前台看不到？

A: 请检查以下几点：

1. 文章状态是否为"已发布"
2. 发布时间是否设置为未来时间
3. 文章是否设置为"私密"
4. 缓存是否已更新

### Q: 如何批量管理文章？

A: 在 [文章管理](Article-Management) 页面：

1. 勾选需要操作的文章
2. 点击批量操作按钮
3. 选择操作类型（删除、修改状态等）

### Q: 支持哪些 Markdown 语法？

A: 支持标准 Markdown 语法，包括：

- 标题、列表、引用
- 代码块（支持语法高亮）
- 表格
- 图片、链接
- 任务列表
- 数学公式（需要配置）

### Q: 如何插入图片？

A: 有以下几种方式：

1. 直接拖拽图片到编辑器
2. 点击工具栏的图片按钮上传
3. 使用 Markdown 语法：`![描述](图片URL)`
4. 粘贴剪贴板中的图片

## 文件上传

### Q: 图片上传失败？

A: 请检查：

1. [上传配置](Upload-Config) 是否正确
2. 文件大小是否超过限制
3. 文件格式是否支持
4. 存储空间是否充足
5. 存储服务是否可用（如使用云存储）

### Q: 支持哪些文件格式？

A: 默认支持：

- 图片：jpg、jpeg、png、gif、webp、svg
- 文档：pdf、doc、docx
- 其他：根据配置可自定义

### Q: 如何配置云存储？

A: 在 [上传配置](Upload-Config) 中选择存储方式：

- 本地存储
- 阿里云 OSS
- 腾讯云 COS
- 七牛云
- AWS S3

## 通知功能

### Q: 邮件通知发送失败？

A: 请检查 [通知配置](Notification-Config)：

1. SMTP 服务器地址是否正确
2. 邮箱账号密码是否正确
3. 是否开启了 SMTP 服务
4. 防火墙是否允许 SMTP 端口

### Q: 支持哪些通知方式？

A: 目前支持：

- 邮件通知
- 微信通知
- 飞书通知
- 站内通知

## AI 功能

### Q: 如何启用 AI 功能？

A: 在 [AI 配置](AI-Config) 中：

1. 选择 AI 服务提供商
2. 配置 API Key
3. 设置模型参数
4. 启用相应功能

### Q: 支持哪些 AI 服务？

A: 目前支持：

- OpenAI（ChatGPT）
- 文心一言
- 通义千问
- 其他兼容 OpenAI API 的服务

### Q: AI 功能消耗如何计费？

A: 根据所选 AI 服务商的计费规则，通常按 Token 数量计费。建议设置使用限额。

## 性能优化

### Q: 如何提升访问速度？

A: 可以采取以下措施：

1. 启用缓存（Redis）
2. 使用 CDN 加速静态资源
3. 优化图片大小
4. 启用 Gzip 压缩
5. 使用数据库索引

### Q: 如何减少服务器负载？

A: 建议：

1. 启用页面缓存
2. 限制并发请求
3. 使用队列处理耗时任务
4. 定期清理日志和临时文件

## 数据备份

### Q: 如何备份数据？

A: 使用 [导入导出](Import-Export) 功能：

1. 导出配置数据
2. 备份数据库
3. 备份上传的文件
4. 定期自动备份

### Q: 如何迁移数据？

A: 迁移步骤：

1. 在旧系统导出数据
2. 在新系统导入数据
3. 迁移上传的文件
4. 更新配置信息

## 安全问题

### Q: 如何防止恶意攻击？

A: 安全措施：

1. 使用强密码
2. 启用 HTTPS
3. 定期更新系统
4. 限制登录尝试次数
5. 配置防火墙
6. 定期备份数据

### Q: 如何防止 SQL 注入？

A: 系统已采取以下措施：

1. 使用参数化查询
2. 输入验证和过滤
3. 最小权限原则
4. 定期安全审计
