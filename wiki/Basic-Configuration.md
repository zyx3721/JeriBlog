# 基础配置

本章节介绍 JeriBlog 的各项基础配置，包括站点信息、上传设置、第三方服务集成等。完成这些配置后，你的博客系统将具备完整的功能。

## 配置概览

| 配置项 | 说明 | 重要程度 |
| --- | --- | --- |
| [基本配置](Basic-Config) | 站点名称、描述、Logo 等基础信息 | ⭐⭐⭐ 必需 |
| [博客配置](Blog-Config) | 博客相关的显示和功能配置 | ⭐⭐⭐ 必需 |
| [上传配置](Upload-Config) | 文件上传方式和存储配置 | ⭐⭐⭐ 必需 |
| [OAuth 配置](OAuth-Config) | 第三方登录（GitHub、Google 等） | ⭐⭐ 推荐 |
| [AI 配置](AI-Config) | AI 功能集成（ChatGPT、文心一言等） | ⭐ 可选 |
| [通知配置](Notification-Config) | 邮件、微信、飞书等通知方式 | ⭐⭐ 推荐 |
| [导入导出](Import-Export) | 数据备份与迁移 | ⭐⭐ 推荐 |

## 配置顺序建议

### 第一步：基础配置（必需）

1. [基本配置](Basic-Config) - 设置站点基本信息
2. [博客配置](Blog-Config) - 配置博客显示选项
3. [上传配置](Upload-Config) - 配置文件上传方式

完成这三项配置后，博客即可正常运行。

### 第二步：功能增强（推荐）

4. [OAuth 配置](OAuth-Config) - 启用第三方登录
5. [通知配置](Notification-Config) - 配置消息通知

这些配置可以提升用户体验和管理效率。

### 第三步：高级功能（可选）

6. [AI 配置](AI-Config) - 集成 AI 功能
7. [导入导出](Import-Export) - 数据备份与迁移

根据实际需求选择性配置。

## 配置入口

所有配置项都可以在管理后台的 **系统管理 → 系统设置** 中找到。

访问路径：`/admin/setting`

## 配置文件

配置信息存储在数据库中，部分敏感信息（如 API Key）建议使用环境变量管理。

## 常见问题

**Q: 配置修改后需要重启服务吗？**  
A: 大部分配置修改后会立即生效，无需重启。但涉及环境变量的修改需要重启服务。

**Q: 配置错误导致无法访问怎么办？**  
A: 可以直接修改数据库中的配置表，或通过 [导入导出](Import-Export) 功能恢复配置。

**Q: 如何备份配置？**  
A: 使用 [导入导出](Import-Export) 功能可以导出所有配置为 JSON 文件。
